    老酒 11/30/2021 git

    # git 常用命令

    git clone  克隆代码地址

git add  index.html  增加一个文件

git commit -m "这里写注释"
git commit --no-verify -m '。。。' 不做代码风格校验
git pull   拉取最新代码 （如果该项目多人协同时，每次push 之前 先拉取一次，避免代码冲突）
git commit -am '。。。' 不存到暂存区直接commit
git push  提交服务器

git status  查看目前的状态

git fetch origin dev（dev为远程仓库的分支名）  把远程分支拉到本地

git checkout -b 分支名字 创建新分支并立即切换到新分支

git checkout -b dev(本地分支名称) origin/dev(远程分支名称) 在本地创建分支dev并切换到该分支

git pull origin dev(远程分支名称) 把某个分支上的内容都拉取到本地

git push origin test:master         // 提交本地test分支作为远程的master分支
git push origin test:test           // 提交本地test分支作为远程的test分支

git push origin :test              // 刚提交到远程的test将被删除，但是本地还会保存的

git push -u origin newBranch // 最后将新分支推送到远端或者说与远端某个分支建立联系


git push --set-upstream origin 分支名 // 将当前分支和线上已有分支建立联系

git branch -m oldname newname // 当前分支重命名
