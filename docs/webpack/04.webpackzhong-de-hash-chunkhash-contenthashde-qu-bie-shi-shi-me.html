    webpack中的hash,chunkhash,contenthash的区别是什么

    vuePress-theme-reco 老酒    2021-12-07 - 2022

    webpack中的hash,chunkhash,contenthash的区别是什么

    老酒 1/10/2022 webpack

    # 背景

    首先来个背景介绍，哈希一般是结合 CDN 缓存来使用的。如果文件内容改变的话，那么对应文件哈希值也会改变，对应的 HTML 引用的 URL 地址也会改变，触发 CDN 服务器从源服务器上拉取对应数据，进而更新本地缓存。

    # 一个简单的webpack配置

    首先我们就只创建一个能编译js的webpack配置，步骤如下：

    1. 创建一个空文件夹名称为webpack-hash(mkdir webpack-hash)
    2. 进入文件夹(cd webpack-hash)执行npm init -y生成package.json
    3. 安装webpack包依赖：npm install webpack webpack-cli
    4. 给pageage.json中的script添加一条命令："build": "webpack --config webpack.config.js"
    5. 添加src目录，里面添加index.js,index2.js并编写其中代码
    6. 在项目根目录中创建webpack.config.js
    7. 执行npm run build命令

    # 1.hash

    我们直接在终端执行npm run build运行结果如下图所示：

    此时index和index2的hash相同，如果我们修改index.js的代码：

    console.log(1234) // 123 => 1234
    1

    重新打包会发现所有的hash都发送了变化，如图所示：

    和上如对比，我们会发现他们的hash并不相同，但是我们只改了index.js的代码，为什么会index2的hash值也改变了呢？

    是因为hash 所有文件哈希值相同，只要改变内容跟之前的不一致；所有哈希值都改变，没有做到缓存意义；所有就要用到chunkhash了。

    # contenthash

    但是当我们一个js文件里面引用了一个css文件，如果我么修改了css文件内的内容，我们css中的内容，会发发现这整个bundle的hash也会发生更新。 我们要引入css，并且要把css提出、压缩生成一个css文件，就要借助一个webpack的插件，叫做MiniCssExtractPlugin,他可以帮我提取css到css文件，并且压缩css。

    第一步先安装css-loader、mini-css-extract-plugin包

    npm install css-loader mini-css-extract-plugin -D
    1

    第二步修改webpack.config.js 如下

    const MiniCssExtractPlugin = require("mini-css-extract-plugin"); // 新增
module.exports = {
  mode: "production",
  entry: {
    index: "./src/index.js",
    chunk1: "./src/index2.js"
  },
  output: {
    filename: "[name].[chunkhash].js"
  },
  module: { // 新增
    rules: [
      {
        test: /\.css$/,
        use: [
          MiniCssExtractPlugin.loader, 
          "css-loader"
        ]
      }
    ]
  },
  plugins: [ // 新增
    // 提取css插件
    new MiniCssExtractPlugin({
      filename: "[name].[chunkhash].css"
    })
  ]
};

    第七步我们再重新npm run build：

    对比两次构建的hash，发现只修改了index.js的文件，但是我们的index.css内容没改变啊，但是打包之后他们的hash都发生了变化，缓存也失效了。如何解决问题呢？

    contenthash 将根据资源内容创建出唯一 hash，也就是说文件内容不变，hash 就不变。

    第八步修改webpack.config.js的配置并且重新npm run build

    const MiniCssExtractPlugin = require("mini-css-extract-plugin");
module.exports = {
  mode: "production",
  entry: {
    index: "./src/index.js",
    index2: "./src/index2.js"
  },
  output: {
    filename: "[name].[chunkhash].js"
  },
  module: { 
    rules: [
      {
        test: /\.css$/,
        use: [
          MiniCssExtractPlugin.loader, 
          "css-loader"
        ]
      }
    ]
  },
  plugins: [ 
    new MiniCssExtractPlugin({
      filename: "[name].[contenthash].css" // 这里改为 contenthash
    })
  ]
};

    我们可以发现，css 文件的 hash 都没有发生改变；直接使用的缓存好的文件即可。

    # 总结

    hash 计算与整个项目的构建相关；只要改变内容跟之前的不一致，所有哈希值都改变，没有做到缓存意义。

    chunkhash 根据不同的入口文件(Entry)进行依赖文件解析、构建对应的 chunk，生成对应的哈希值；

    contenthash 计算与文件内容本身相关。只要当前文件的内容不变那么hash值就不会改变。
