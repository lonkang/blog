    vue模板编译都做了什么

    老酒 1/27/2022 vue

    # vuejs模板编译分为三个部分：

    1. 将html模板编译为ast对象
    2. 优化，遍历 AST，为每个节点做静态标记，标记其是否为静态节点，然后进一步标记出静态根节点，这样在后续更新的过程中就可以跳过这些静态节点了；标记静态根用于生成渲染函数阶段，生成静态根节点的渲染函数
    3. 从ast生成render函数和staticRenderFns数组(存放了所有静态节点的渲染函数)

    # 1.vue模板编译的解析流程，是怎么将 html 字符串模版变成 AST 对象的？

    • 遍历html模板字符串，通过正则表达式来匹配"<"
    • 跳过一些不需要处理的标签，比如：注释标签，条件匹配标签，Doctype

      整个模板解析的核心就是处理开始标签和结束标签以及文件标签

    • 解析开始标签
      • 得到一个对象，包括 标签名（tagName）、所有的属性（attrs）、标签在 html 模版字符串中的索引位置
      • 再次处理上次得到的attr属性，将其变成[{name: attrName, value: attrVal, start: xx, end: xx}]
      • 然后标签名，属性对象，和当前元素的父亲元素生成ast对象；ast其实就是一个普通的js对象，然后通过key，value的形式记录了元素的一些信息
      • 然后进一步处理ast上面的一些指令，例如：v-pre，v-for，v-if，v-once，并将处理结果放到ast对象上
      • 如果是一元标签就自动执行闭合标签的逻辑，如果不是一元标签就将处理完的ast对象存放到stack数组中
      • 处理完之后会截断html字符串，将已经处理掉的字符串戒掉
    • 解析闭合标签
      • 如果匹配到结束标签，就从stack数组中拿出最后一个元素，它和当前匹配到的结束标签是一对的。如果没找到的话就报错。
      • 然后再次处理开始标签上的属性，这些属性和前面处理的不一样，比如key，ref，插槽，class，style，动态组件等，并将处理完的结果放到ast对象上
      • 然后将当前元素和父元素产生联系，给当前元素的 ast 对象设置 parent 属性，然后将自己放到父元素的 ast 对象的 children 数组中
    • 解析文本标签
      • 静态标签分为两种：type为 2 表示是表达式，type为 3 表示是纯文本。
      • 如果标签是<div></div>中，那是表达式
      • 如果标签是这样的<div>123</div>中，那123是纯文本标签
      • 然后添加到父节点中
    • 最后遍历完整个 html 模版字符串以后，返回 ast 对象

    # 2.优化ast

    为什么要有优化过程，因为我们知道 Vue 是数据驱动，是响应式的，但是我们的模板并不是所有数据都是响应式的，也有很多数据是首次渲染后就永远不会变化的，那么这部分数据生成的 DOM 也不会变化，我们可以在 patch 的过程跳过对他们的比对。

    静态标记的过程分为两个步骤，标记静态节点和静态根节点

    # 标记静态节点我们首先得知道什么是静态节点

    • 如果是表达式，就是非静态；
    • 如果是纯文本，就是静态；
    • 对于一个普通元素，如果有 pre 属性，那么它使用了 v-pre 指令，是静态，否则要同时满足以下条件：没有使用 v-if、v-for，没有使用其它指令（不包括 v-once），非内置组件，是平台保留的标签，非带有 v-for 的 template 标签的直接子节点，节点的所有属性的 key 都满足静态 key；这些都满足则这个 AST 节点是一个静态节点。

    # 标记静态节点

    • 通过递归的方式，标记所有的元素节点
    • 如果节点本身是静态节点，但是存在非静态节点的子节点，那么修改节点为非静态节点

    # 标记静态根节点

    • 如果节点本身是静态节点 && 而且有子节点 && 而且子节点不只是一个文本节点，则标记为静态根节点，否则标记为非静态根节点

    • 如果节点本身不是静态根节点，则递归的遍历所有子节点，在子节点中标记静态根
